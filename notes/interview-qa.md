## ❓ 高频面试题

### 1. 黑盒测试方法有哪些？
- **等价类划分**：将输入域划分为有效/无效等价类
- **边界值分析**：测试边界值（min, min+1, max-1, max）
- **场景法**：模拟用户操作流程
- **判定表**：多条件组合场景
- **错误推测法**：基于经验推测易出错点

### 2. Bug的生命周期？
```
新建(New) → 确认(Open) → 修复(Fixed) → 验证(Verified) → 关闭(Closed)
                              ↓
                          无法修复 → 拒绝(Rejected)
                              ↓
                          再次出现 → 重新打开(Reopened)
```

### 3. 什么是P0/P1/P2/P3？
- **P0**：核心功能，阻塞测试，必须立即修复
- **P1**：主要功能，严重影响用户使用
- **P2**：一般功能，有替代方案
- **P3**：轻微问题，建议性修改

### 4. HTTP常用状态码？
- `200 OK` — 请求成功
- `201 Created` — 创建成功
- `400 Bad Request` — 请求参数错误
- `401 Unauthorized` — 未认证
- `403 Forbidden` — 无权限
- `404 Not Found` — 资源不存在
- `500 Internal Server Error` — 服务器错误

### 5. GET和POST的区别？
| | GET | POST |
|------|------|------|
| 参数位置 | URL中 | 请求体中 |
| 安全性 | 低（URL可见） | 高 |
| 长度限制 | 有限制 | 无限制 |
| 缓存 | 可缓存 | 不可缓存 |
| 幂等性 | 幂等 | 非幂等 |

### 6. Cookie和Session的区别？
- **Cookie**：存储在客户端，不安全，有大小限制
- **Session**：存储在服务端，安全，依赖Cookie传递SessionID

### 7. 如何定位一个Bug？
1. 复现Bug，确认现象
2. 使用F12开发者工具查看网络请求和报错
3. 检查后端日志（Linux: `tail -f` / `grep`）
4. 排查数据库数据是否正确
5. 缩小范围，定位到具体模块/代码行
